Introduction¶

One industry that has long thrived around the world of professional sports is that of sports betting. With the 2018 overturn of the Professional and Amateur Sports Protection Act of 1992 and the advent of the internet, sports betting has recently become more legal and more accessible than ever before in the United States.

This is a hugely important industry, both for the companies racing to make money from bettors and for the millions of Americans spending more than $150 billion every year gambling on sports. As with any type of gambling, the only thing you can be sure of is that the house will always win. But the ease and popularity of sports betting along with news of recent wagers with enormous payouts, it's natural to ask:

Is it possible to outsmart the crowd (and get rich) by using a model to pick which bets to take?

We decided to perform our analysis on the National Football League, as football is the sport with the largest betting market outside of European football (source). There are many ways to place bets on sports, but we chose to focus on two of the most common: Point Spread Betting and Over Under Betting.

An Introduction to Point Spread and Over Under Betting¶

(adapted from a more thorough explanation at OddsShark)

In a point spread bet, oddsmakers set a number, called the "spread" for a given matchup that represents a predicted margin of victory for the favored team. Bets are then placed on which team will "cover the spread": for the underdog, this means losing by a smaller margin than the spread or winning; for the favorite, it means winning by more than the spread. Because almost any matchup has a team that is favored to win, adding a point spread evens out the interest that bettors have in wagering for either side.

As an example, here is a point spread for a recent game between the Miami Dolphins and the New York Jets:

Miami +5.0

New York -5.0

New York was the 5-point favorite, meaning they needed to win the game by 5 points or more to win the bet. As the underdog, Miami needed to either lose by a margin of less than 5 or win the game to win the bet.

The final score of the game was New York 22-21 Miami. Because the margin was smaller than 5 points, bets placed on New York for this matchup lost even though New York won the game. Miami "covered the spread" because they remained within the margin given by the point spread.

For the same game, oddsmakers can also set an over under figure:

New York vs Miami O/U 43.5

Taking the same New York 22-21 Miami scoreline, the total points scored is 43 points. Bets placed on the Under would then win, because the total score was under the figure provided by oddsmakers.

Sometimes when many more bets are being placed for a certain team, sportsbooks will adjust a spread leading up to a game in order to encourage more bets in a certain direction (and ensure that for any result they will make money). If we create a model that predicts NFL margins better than the books, we can take advantage of matchups with underdogs who in reality have a better chance than their spreads reflect.

The success of such a model would be a huge win for data science. It would mean that the NFL betting system can be broken by a well-informed bettor who could then consistently beat other bettors and the house.

Now, we can define and apply functions to change the values in the "Result," "OU," and "ATS" columns, respectively representing the result of selecting the home team to win the game, selecting the home team to beat the spread, and predicting that the total points scored will be greater than the over/under value. We turn these values into ones and zeros, a better labeling system for numeric and logistic analysis later.

In [14]:
# A function which returns 0 if the result was a loss for the home team, or 1 if it was a win for the home team
def label_result (row):
    if pd.to_numeric(row['Home Score']) < pd.to_numeric(row['Away Score']) :
        return 0
    if pd.to_numeric(row['Home Score']) > pd.to_numeric(row['Away Score']) :
        return 1
    #To avoid errors, a -1 is returned if the data is misread, which would alert us to any problems through errors later on
    return -1

# A function which returns 0 if the under was achieved, or 1 if it was not achieved
def label_OU (row):
    if pd.to_numeric(row['Point Total']) < pd.to_numeric(row['Total']) :
        return 0
    if pd.to_numeric(row['Point Total']) > pd.to_numeric(row['Total']) :
        return 1
    #To avoid errors, a -1 is returned if the data is misread, which would alert us to any problems through errors later on
    return -1

# A function which returns 0 if the home team did not cover the spread, or 1 if ithe home team did cover the spread
def label_spread (row):
    if pd.to_numeric(row['Home Score']) + pd.to_numeric(row['Home Spread']) < pd.to_numeric(row['Away Score']) :
        return 0
    if pd.to_numeric(row['Home Score']) + pd.to_numeric(row['Home Spread']) > pd.to_numeric(row['Away Score']) :
        return 1
    #To avoid errors, a -1 is returned if the data is misread, which would alert us to any problems through errors later on
    return -1

# We apply the function to each row, and store the returned results in the appropriate columns,
# replacing the original letter-based representation
all_data['ATS'] = all_data.apply (lambda row: label_spread(row), axis=1)
all_data['OU'] = all_data.apply (lambda row: label_OU(row), axis=1)
all_data['Result'] = all_data.apply (lambda row: label_result(row), axis=1)

The data above gives us a test of the accuracy of models through a K-fold test, which essentially splits the data into k random sets, trains the data on k-1 sets, and applies it to the unused set to check for accuracy. It does k times, cycling through each set as the testing set, and using all combinations of k-1 sets as training sets. We then take the mean of these scores, and plot the above. Due to the nature of this test (splitting randomly), we cannot give any analysis of the data, as with each run of the program it is subject to change, sometimes dramatically.

For T-tests, we have a null hypothesis, in this case that the KFold accuracy tests these models produce come from different samples. For the most part, there seems to be little reason to reject the null hypothesis, as the chances two sets of accuracy scores came from the same sample is never low enough to raise doubt. Therefore, functionally, we cannot reject the idea that these models are very similar.

Coefficient P-Value¶

Because we these models are similar enough that we fail to reject that their accuracy scores could come from the same population, and due to the great simplicity of testing logistic regression coefficient p-values, we decided to use our logistic model to get an idea of the importance of our input variables. We do this by calculating the p-value of each coefficient, with the null hypothesis representing that the coefficient is zero.

Conclusion¶

In this project, we set out to use statistics and machine learning to gain the upper hand on Vegas and the world of sports betting. We hoped to find a space between the initial projections by oddsmakers and the betting habits of the public, in order to turn a profit. However, despite our best efforts, even our most successful models provided us with results marginally better than flipping a coin and saying your prayers. However, our inability to draw create a successful model does not serve as an indictment of the ability to use stats in sports betting. We have only explored a narrow slice of the NFL betting world, and with a select few variables. It is very possible that there is a metric which would provide newfound accuracy to our models. Furthermore, it is possible, likely even, that with more data, we would be able to make better selections.

In the end, it seems that the old adage is true; The house always wins. For now.
